Sử dụng Test Mode để giả lập môi trường development của bạn.

Test Mode của SimplifyTrip cho phép bạn tích hợp hệ thống của mình mà không cần thực hiện các khoản phí hoặc thanh toán thực tế.
Test Mode là môi trường giả lập mô phỏng việc tạo các giao dịch thực mà không ảnh hưởng đến các giao dịch thực hoặc dòng tiền thực.
Chúng tôi khuyên bạn nên sử dụng Test Mode để đảm bảo chất lượng (QA) trong quá trình phát triển hệ thống của mình.

### Test Mode
Ở chế độ thử nghiệm, bạn có thể mô phỏng các giao dịch giả lập nhằm đảm bảo tích hợp của bạn hoạt động chính xác.
Tính năng này giúp xác định bất kỳ lỗi hoặc sai sót nào trong quá trình triển khai hệ thống của bạn với SimplifyTrip trước khi bạn đưa vào hoạt động với các giao dịch thực tế.

Sau khi tạo tài khoản SimplifyTrip, bạn có thể tìm thấy một bộ Test API Key thử nghiệm trong SimplifyTrip Dashboard . Bạn có thể sử dụng các Test API Key này để tạo và truy xuất dữ liệu mô phỏng bằng cách gửi yêu cầu đến SimplifyTrip API.
Để bắt đầu những giao dịch thực, bạn cần **kích hoạt tài khoản** của mình , tắt Test Mode và sử dụng **Live API Key** trong tích hợp của bạn.

### Chế độ Test Mode so với chế độ Live Mode

Tất cả các yêu cầu API SimplifyTrip đều xảy ra ở chế độ Test Mode hoặc chế độ Live Mode. Các đối tượng API ở chế độ này không thể truy cập được ở chế độ kia. Ví dụ,  ở chế độ Test Mode các Tài khoản không thể thực hiện thanh toán ở chế độ Live Mode.

Nút chuyển đổi chế độ **Test Mode** trong Bảng điều khiển không ảnh hưởng đến mã tích hợp của bạn. API Key chế độ Test Mode và chế độ Live Mode của bạn mới ảnh hưởng đến hành vi của mã. API Key bạn sử dụng để xác thực yêu cầu sẽ được dùng để xác định xem yêu cầu đó là chế độ Live Mode hay Test Mode. Api Key chế độ thử nghiệm có tiền tố `pk_test_` và Api Key chế độ live có tiền tố `pk_live_`.
