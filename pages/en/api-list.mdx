# Danh sách các SimplifyTrip APIs hỗ trợ.

API SimplifyTrip được tổ chức xấy dựng dựa trên [REST](https://en.wikipedia.org/wiki/REST). API của chúng tôi là các resource-oriented URLs có thể dự đoán được, chấp nhận [form-encoded](https://en.wikipedia.org/wiki/POST_(HTTP)#Use_for_submitting_web_forms) request bodies, trả về các phản hồi được mã hóa theo [JSON-encoded](https://www.json.org/json-en.html) và sử dụng các mã phản hồi HTTP chuẩn, authentication, and verbs.

Bạn có thể sử dụng API SimplifyTrip ở **Test Mode**, điều này không ảnh hưởng đến dữ liệu thực tế của bạn và có thể tương tác với các Tài khoản ảo có sẵn. API Key bạn sử dụng để xác thực yêu cầu sẽ được dùng để xác định xem yêu cầu đó là chế độ `LIVE_MODE` hay `TEST_MODE`.

#### Authentication
SimplifyTrip API sử dụng API Key để xác thực yêu cầu truy cập của bạn. Bạn có thể xem và quản lý API Key của mình trong <a href={process.env.NEXT_PUBLIC_PARTNERS_URL + '/developers'} className="underline" target='_blank'>Api Keys Settings</a>.

Api Key chế độ thử nghiệm có tiền tố `pk_test_` và Api Key chế độ live có tiền tố `pk_live_`.

Secret Key chế độ thử nghiệm có tiền tố `sk_test_` và Secret Key chế độ live có tiền tố `sk_live_`.

API Key và Secret Key của bạn có nhiều đặc quyền, vì vậy hãy đảm bảo giữ chúng an toàn! Không chia sẻ API Key và Secret Key của bạn ở những nơi công khai như GitHub, mã phía máy khách, v.v.

Tất cả các yêu cầu API phải được thực hiện qua HTTPS . Các cuộc gọi được thực hiện qua HTTP thông thường sẽ không thành công. Các yêu cầu API không xác thực cũng sẽ không thành công.

#### Requirements
- **Api Key và Secret Key**

	Mọi request tới SimplifyTrip API đều PHẢI bao gồm khóa Api Key và Secret Key trong headers.

```json
X-Api-Key: "pk_live_xxxxxxxxxxxxxxxxxx"
Authorization: "Bearer sk_live_xxxxxxxxxxxxxxxxxx"
```

- **Content Type**

	Content type của request PHẢI là `application/json`.

```json
Content-Type: "application/json"
```
