# Supported SimplifyTrip APIs

SimplifyTrip APIs are built on [REST](https://en.wikipedia.org/wiki/REST). Endpoints use predictable resource-oriented URLs, accept request payloads, return [JSON](https://www.json.org/json-en.html), and use standard HTTP status codes.

You can use SimplifyTrip APIs in **Test Mode** without affecting real data. The API key used in authentication determines whether the request runs in `LIVE_MODE` or `TEST_MODE`.

#### Authentication
SimplifyTrip API uses API keys to authenticate requests. You can manage your API keys in <a href={process.env.NEXT_PUBLIC_PARTNERS_URL + '/developers'} className="underline" target='_blank'>API Keys Settings</a>.

Test API keys start with `pk_test_`, live API keys start with `pk_live_`.

Test secret keys start with `sk_test_`, live secret keys start with `sk_live_`.

Keep keys secure. Do not expose API Key or Secret Key in public places such as GitHub or client-side code.

All API requests must be made over HTTPS.

#### Requirements
- **Api Key and Secret Key**

	Every request to SimplifyTrip API MUST include Api Key and Secret Key in headers.

```json
X-Api-Key: "pk_live_xxxxxxxxxxxxxxxxxx"
Authorization: "Bearer sk_live_xxxxxxxxxxxxxxxxxx"
```

- **Content Type**

	The request content type MUST be `application/json`.

```json
Content-Type: "application/json"
```
